File / Folder Structure

Introduction

A specific methodology is employed to the file structure. The primary goal of the structure is consistency between projects, and ease of scalability and duplication. SP lends itself well to templating. Using the same starting files, you can easily port the methodology to multiple projects. Furthermore, it helps to avoid order of operation errors and other common pitfalls.

File Descriptions

File Name Description
____README_src.md Contains detailed documentation about the source code, including setup instructions, dependencies, and usage examples.
__syntax_references.txt Reference guide for Python syntax and Extron ControlScript specific syntax used throughout the project.
__main.py The primary entry point for the application. Contains initialization code and the main program loop.
__mainFinal.py The finalized version of the main program, optimized and ready for production deployment.
fn_coreFunctionality.py Contains core functions for the user interface, including button handlers, page navigation, and display updates.
fn_Fox3.py Functions specific to the Fox3 series hardware, including specialized control and configuration routines.
ModuleSupport.py Provides support utilities for Extron modules, including module loading, version checking, and compatibility functions.
____Framework_History.md Detailed changelog documenting the version history of the SP Framework, including features, bug fixes, and improvements.
____README_sp_Framework.md Comprehensive documentation for the SP Framework, including architecture overview, usage guidelines, and best practices.
__sp_bootAgent.py Manages the framework boot sequence, ensuring proper initialization of components in the correct order.
__sp_fileOrder.py Controls the order in which files are loaded and processed, preventing dependency issues and ensuring proper initialization.
__sp_framework.py The core framework file that provides the main functionality and interfaces for the SP Framework.
__sp_libPYTZ_2020.py Timezone library for handling time zone conversions, daylight saving time adjustments, and time-related operations.
__sp_services.py Implements various services used by the framework, such as logging, error handling, and communication services.
__sp_textTemplates.py Manages text templates for generating consistent messages, notifications, and user interface text.
__sp_timingAndClocks.py Provides utilities for timing operations, scheduling, and clock display functions.
__sp_tools.py Collection of general-purpose tools and utilities used throughout the framework.
__sp_variables.py Defines and manages global variables used by the framework and applications built on it.
extr_matrix_Fox3_Series_v1_6_0_0.pdf Official documentation for the Fox3 Series matrix switchers, including specifications, installation guides, and programming references.
extr_matrix_VN_Matrix_v1_8_0_0.pdf Official documentation for the VN Matrix series, covering features, setup procedures, and integration guidelines.
extr_sm_SMD101_SMD202_v1_11_9_0.pdf Official documentation for the SMD101 and SMD202 streaming media decoders, including technical specifications and configuration options.
extr_matrix_Fox3_Series_v1_6_0_0.py Python module for controlling and interfacing with Fox3 Series matrix switchers, implementing the API defined in the corresponding documentation.
extr_matrix_VN_Matrix_v1_8_0_0.py Python module for controlling VN Matrix devices, providing programmatic access to all device functions and settings.
extr_sm_SMD101_SMD202_v1_11_9_0.py Python module for controlling SMD101 and SMD202 streaming media decoders, with functions for configuration, monitoring, and control.
__sandbox_moduleLibrary.py Library of modules available within the sandbox environment, providing a controlled set of components for sandbox applications.
__sandbox_tools.py Collection of tools specific to the sandbox environment, including testing, debugging, and development utilities.
__sandbox_variables.py Defines and manages variables used within the sandbox environment, including configuration settings and state information.
__sandbox_devices.py Defines the devices available within the sandbox environment, including their capabilities, limitations, and configuration options.
__sandbox_devices_csv.py Provides functions for importing and exporting device configurations using CSV files, enabling easy bulk configuration.
__sandbox_module_components.py Defines the components that make up modules within the sandbox, including their interfaces, behaviors, and interactions.
__sandbox_routing_csv.py Provides functions for importing and exporting routing configurations using CSV files, enabling flexible signal routing setup.

What is a Sandbox?

Introduction

In the SP Framework, a sandbox is a controlled environment that defines the boundaries and capabilities of a control system. It provides a structured way to specify what hardware devices are available, how they connect to each other, and what functionality is supported.

The sandbox concept establishes the "walls" or limits of what your control system can do, defining parameters such as the minimum and maximum number of displays, the available inputs and outputs, and the supported device types.

Key Components of a Sandbox

Based on the files found in the NIWC_GOFO project, a sandbox in the SP Framework consists of the following components:

Core Files

  • __sandbox_variables.py - Contains the sandbox_config dictionary that holds configuration parameters for the sandbox environment.
  • __sandbox_tools.py - Provides utilities for sandbox creation and management.
  • __sandbox_moduleLibrary.py - Contains the library of modules available within the sandbox environment.

Device Management

  • __sandbox_devices.py - Defines the devices available within the sandbox environment.
  • __sandbox_devices_csv.py - Provides the import_devicesConfig_CSV function for importing device configurations from CSV files.
  • __sandbox_module_components.py - Defines the components that make up modules within the sandbox.

Routing Management

  • __sandbox_routing_csv.py - Provides the import_matrixConfig_CSV function for importing routing configurations from CSV files.

Sandbox Implementation Workflow

Based on the structure of the sandbox files, implementing a sandbox follows this workflow:

1. Project Scope Definition

  • Define Room Type - Categorize spaces based on functionality and requirements
  • Determine Equipment Type - Identify the appropriate technology solutions for each space

2. Determine Equipment Footprint

  • Models of displays - Select appropriate display technologies and sizes
  • Cameras, Codecs - Choose video capture and processing equipment
  • Matrix Switchers - Determine signal routing and distribution needs
  • DSP's - Select audio processing solutions

3. Determine Sandbox Walls

  • Min-Max displays - Define the range of supported display configurations
  • Min-Max Inputs - Establish input capacity requirements and limitations
  • Supported Classifications - Identify security and compliance requirements

4. Determine Control Footprint / Deltas

  • Cameras - Define camera control requirements and presets
  • Codecs, Blu-ray - Configure media playback and conferencing controls
  • Audio, Time Zone Clocks - Set up audio management and time displays
  • Messaging - Implement notification and communication systems

Tiered Sandbox Management

The SP Framework provides a tiered approach to sandbox management:

Tier III - Programmer

Programmers are responsible for the initial creation and architecture of sandbox environments.

  • Create initial sandbox - Establish the foundation and architecture of the sandbox environment
  • Minimal follow up support - Provide specialized assistance for complex issues

Required Certification: Extron Authorized Programmer

Tier II - AV Maintenance

AV Maintenance personnel handle the day-to-day management of sandbox environments.

  • Create / edit sandbox config files - Modify configuration settings to adapt the sandbox to specific requirements
  • Main follow up support - Serve as the primary support contact for ongoing sandbox management

Required Certification: Extron AV Associate

Tier I - AV Tech / End User

End users and AV technicians interact with the sandbox through intuitive interfaces.

  • Basic Diagnostic tools available on the glass - Access to user-friendly diagnostic interfaces
  • Provide valuable feedback to AV Maintenance - Report issues and suggest improvements based on practical usage

Recommended Certification: Extron AV Principles

Conclusion

The sandbox concept is a fundamental aspect of the SP Framework, providing a structured approach to creating, configuring, and managing control systems. By defining clear boundaries and capabilities, sandboxes ensure that control systems remain scalable, maintainable, and consistent across different installations.

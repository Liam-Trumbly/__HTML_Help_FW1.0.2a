Host Management

The SP Framework provides a structured approach to managing AV control systems, ensuring consistency and scalability. This section covers essential functions and their descriptions, categorized for ease of reference.

Host Management provides an easy way to build Host devices from a string without the need for manual editing of hard code. At its simplest level, Host Devices are hardware devices that host physical interfaces and ports such as COM, IR, Flex I/O, EthernetClientServers, and others. The most common Host Devices are the IPCP Processors themselves. Traditional instantiation methods for Host Devices are defined in the Extron ControlScript device Package.

#### ~ ControlScript ###################################################################################################
from extronlib.device import SPDevice, ProcessorDevice

Function Description
register_host Creates a global tie-in for processors and SPDs.
gHost Retrieves a host from the host database.
getDefaultHost Retrieves the default UIHost from the configuration.
registerAllHosts Registers all hosts defined in the sandbox configuration.
anyUI_Host Retrieves the UI host from the tlpDB based on the provided ui_name.

register_host

The register_host function allows global registration of a host, making it accessible throughout the system. It is primarily used for processors but also supports Secure Platform Devices (SPDs). 

register_host(host_name, device_class='Processor', description='', header='', text_return=None)

Parameters

Parameter Type Description
host_name str A friendly name used to identify the host. Example: "Processor_Main".
device_class str The type of device to register. Options:
  • 'Processor' (default) – Represents a processing unit.
  • 'SecurePlatform' – Represents a Secure Platform Device.
description str An optional description of the host's purpose. Example: "Handles main processing tasks."
header str Additional information to display before registration details. Example: "System Initialization".
text_return function An optional callback function for handling returned messages.

Return Value

This function does not return a value directly. However, if text_return is provided, the function will pass status messages to it.

Example Usage


## ! Register a processor
# Simplified Version
register_host("Processor_Main", "Processor", "Handles main processing tasks")

# Using Keywords
register_host(host_name="Processor_Main", device_class="Processor", description="Handles main processing tasks")

# Register a Secure Platform Device
register_host("SPD_Temperature_Sensor", "SecurePlatform", "Monitors temperature for secure platforms")

# Utilize text_return
register_host("Processor_Main", "Processor", "Handles main processing tasks", text_return=print)
## This example will take in the function print and pass status messages to it.

Error Handling

If an invalid device_class is provided, an error message will be generated and logged.

gHost

The gHost function retrieves a host object from the host database. If the requested host is not registered, it raises an error. 

gHost(host_to_retrieve)

Parameters

Parameter Type Description
host_to_retrieve str The name of the host to retrieve.

Return Value

Returns the registered host object.

Default location in Sandbox Config


sandbox_config['hosts'] = {}
sandbox_config['hosts']['All'] = \
    [
        ## ! ADD PROCESSORS, EXP, and SPD Here ##
        ('Processor_Main', 'Processor', 'Main Processor for all control ports'),
        # ('Power_Ctrl', 'SecurePlatform', 'Four Port Power Control'),
        # ('EXP_S5', 'SecurePlatform', 'Five Port Serial Expansion')
    ]
sandbox_config['hosts']["Default"]= "Processor_Main"

Example Usage


## ! Register a processor
#! Get the host object
globalHost = gHost("Processor_Main")

#! Print the host's model name and IP address
print(globalHost.ModelName)
print(globalHost.IPAddress)

#! Reboot the host
globalHost.Reboot()

Error Handling

Raises a RuntimeError if the host is not found.

getDefaultHost

The getDefaultHost function retrieves the default UIHost from the configuration. It can return the UIHost as an alias or as a host object. 

getDefaultHost(return_as_alias=False)

Parameters

Parameter Type Description
return_as_alias bool If True, returns the alias identifier; otherwise, returns the host object.

Return Value

Returns the UIHost alias (str) or the host object.


#! Get the default host object
defaultHost = getDefaultHost()

#! Print the host's model name and IP address
print(defaultHost.ModelName)
print(defaultHost.IPAddress)

#! Reboot the host

#! Get the host object alias

print(defaultHostAlias)

Error Handling

Logs an error and returns None if no UIHost is found.

anyUI_Host

The anyUI_Host function retrieves the UI Host from the TLP Database using the ui_name. Remember that there is a distinction between hosts (which represent Processors, SPDevices, etc.) and UI_Hosts (which represent Touch Link Panels and other User Interface Devices. If wildcard is True, it will search for multiple UI_Hosts and will return a list of UI_Host objects whois alias begins with the text set in the ui_name argument. If wildcard is False, it will only search for a single UI_Host and return it as a UI_Host object (not a list). 

anyUI_Host(ui_name, wildcard=False)

Parameters

Parameter Type Description
ui_name str The name of the UI to be retrieved.
wildcard bool If True, search using wildcard to match multiple keys.

Example Usage


## ! Register a processor
# Simplified Version
register_host("Processor_Main", "Processor", "Handles main processing tasks")

# Using Keywords
register_host(host_name="Processor_Main", device_class="Processor", description="Handles main processing tasks")

# Register a Secure Platform Device
register_host("SPD_Temperature_Sensor", "SecurePlatform", "Monitors temperature for secure platforms")

# Utilize text_return
register_host("Processor_Main", "Processor", "Handles main processing tasks", text_return=print)
## This example will take in the function print and pass status messages to it.

Return Value

uiHost or list: The matched UI host object(s) from tlpDB.

Error Handling

This function raises a system error flag if no matching key is found.


Global Functions

Global functions are used to access functions from any function file without needing to import modules.


Function Description

gFunc Call a global function with the given ID and pass in arguments and Keyword args.
gFuncLoader Allows you to 'load' a global function from one file (including args & kwargs), but execute it in another.

register_gFunc

The register_gFunc function creates a global Tie-in for functions to recall from any Function File

register_gFunc(function, description='')

Parameters

Parameter Type Description
function callable The function to register
description str Optional description of the function

Return Value

This function does not return a value but logs messages regarding registration status.

gFunc

The gFunc function is used to call a global function with the given ID and arguments.

gFunc(function_id, *args, **kwargs)

Parameters

Parameter Type Description
function_id str The ID of the global function to call
*args Positional arguments for the global function
**kwargs Keyword arguments for the global function

Return Value

The result of the global function call.

Error Handling

There are three different types of errors for this function:
- KeyError: If the global function ID is not registered.
- TypeError: If the function ID is not a string.
- Exception: If the global function call raises an exception.

gFuncLoader

The gFuncLoader function allows you to 'load' a global function from one file (including args & kwargs), but execute in another. For example: if you are naming a preset from the signage page, you can prepare a global function in signage, but not run the global function until you hit 'accept' in the keyboard file. 

gFuncLoader(gFunc_name, *args, **kwargs)

Parameters

Parameter Type Description
gFunc_name str The name of the global function to load
*args Positional arguments for the global function
**kwargs Keyword arguments for the global function

Code Example


class gFuncLoader:
"""
BETA---

Allows you to 'load' a global function from one file (including args & kwargs), but execute in another.
For example:  if you are naming a preset from the signage page, you can prepare a global function in signage,
but not run the global function until you hit 'accept' in the keyboard file.
"""
    def __init__(self, gFunc_name, *args, **kwargs):
        self.gFunc_name = gFunc_name
        self.args = args
        self.kwargs = kwargs

anyBtn

The anyBtn function Retrieves a list of buttons from the tlpDB based on the ui_object_name provided. Returns a list of all button objects in the UI database matching the given name.

anyBtn(ui_object_name, wildcard=False, UIHosts=None)

Parameters

Parameter Type Description
ui_object_name str The identifier or part of the button name to search for. When using wildcard matching, this string should appear at the beginning of the button names.
wildcard bool, optional If True, enables a wildcard (substring) mode where buttons starting with ui_object_name are matched. Defaults to False (exact matching).
UIHosts list or OrderedDict, optional A list of UI host identifiers to restrict the search results. If provided, only buttons associated with the specified UI hosts are considered. If None, buttons from all UI hosts are included.

Return Value

list: A list of button objects that match the search criteria. If no buttons are found, an empty list is returned.

Error Handling

KeyError: Caught and logged when an exact match is attempted but the ui_object_name key does not exist in the UI database.
Exception: Any other errors encountered during the search are caught and logged with details.

Example Usage


    # Retrieve a list of exit buttons used for idle state event handling
    exit_buttons = anyBtn('btn_exit_idle')

    # Register an event trigger using the returned button object(s):
    @event(anyBtn('btn_exit_idle'), 'Pressed')
    def btn_exit_idle_event(button, state):
        dvPressTrace(button, state)
        MultiShowPage('01_Main')

The anySlider function Searches the UI database for slider objects that match the specified criteria and returns a list of matching slider objects.

anySlider(object_name, wildcard=False, UIHosts=None)

Parameters

Parameter Type Description
ui_object_name str The slider name or substring to search for within the slider names.
- If wildcard is False, object_name must match a slider name exactly.
- If wildcard is True, object_name can be any substring of the slider name.
wildcard bool, optional When set to True, enables substring matching for slider names. Defaults to False.
UIHosts list or OrderedDict, optional A list of UI host identifiers on which to filter the results. If None, no filtering is applied. It can be used to limit the search to specific UI hosts.

Return Value

Exception: Any other exceptions encountered during the search process will trigger a system error message with details of the exception.

Behavior

Behavior:
1. Retrieves the UI hosts via the get_ui_hosts() function if UIHosts is not provided.
2. If wildcard matching is enabled:
- Iterates through all slider names in __sp_var.uiDB['sld'].
- For each slider name that contains the provided object_name as a substring, iterates through all associated slider objects,
adding them to the result list if their UI host is in the filtered UIHosts (if provided).
3. If wildcard matching is disabled:
- Attempts to retrieve the slider dictionary directly using object_name as the key.
- Iterates through the associated slider objects, adding each to the result list if its UI host matches one of the filtered UIHosts.
4. If no slider objects are found, a debug trace is generated to log that the returned slider list is empty.
5. The function finally returns the list of matching slider objects.

formatTable_listOfDicts

Formats a list of dictionaries into a table-like string representation, handling multi-line headers with various newline conventions (CR, LF, CRLF). Optionally obscures cell text based on keywords in headers. 

formatTable_listOfDicts(data, custom_order=None, title='', return_max_len=False, keywords_to_obscure=None)

Parameters

Parameter Type Description
data list A list of dictionaries to be formatted into a table.
custom_order list, optional A list of keys that defines the order of columns. If not provided, headers are extracted from the first dictionary.
title str, optional A title to display above the table.
return_max_len bool, optional If True, also returns the maximum length of the table's separator line. Default is False
keywords_to_obscure list, optional A list of keywords. If any keyword is found in a header (case-insensitive), the corresponding cell text will be obscured with asterisks.

Returns

str or tuple: The formatted table as a string. If return_max_len is True, returns a tuple containing the formatted table and the length of the separator line.

Example Usage


>>> data = [
...     {"Name": "Alice", "Age": 30, "Secret": "Loves Python"},
...     {"Name": "Bob", "Age": 25, "Secret": "Hates bugs"}
... ]
>>> custom_order = ["Name", "Age", "Secret"]
>>> table, sep_length = formatTable_listOfDicts(data, custom_order=custom_order, title="User Info", return_max_len=True, keywords_to_obscure=["Secret"])
>>> print(table)
User Info
Name  | Age | Secret
-------------------------
Alice | 30  | ************
Bob   | 25  | ************
-------------------------
>>> print("Separator length:", sep_length)

elipser

Truncates long strings by removing characters from the left and prepending an ellipsis (or defined padding) so the result fits a target width 

elipser(text, width, pad='...', forceWidth=False)

Example Usage

>>> elipser('abcdefghijklmnop', 8)
'...lmnop'

>>> elipser('abc', 8)
'abc'

>>> elipser('abc', 8, forceWidth=True)
'abc     '  # padded to width 8

If len(text) > width, the function strips characters from the left until the remaining text fits width - len(pad), then prepends pad. With forceWidth=True the result is padded to exactly width.

get_file_class

This function takes a string representing a file class ('file' or 'rfile') and returns the associated class object (`File` or `RFile`). The input string is case-insensitive. If an invalid file class string is provided, it logs a system error using `flag_system_error` and does not return a value (implicitly returns None after the error logging). 

get_file_class(_file_class)

Parameters

Parameter Type Description
_file_class str The string representation of the desired file class (e.g., 'file', 'rfile').

Returns

object: The class object corresponding to the input string (`File` or `RFile`). Returns None if the input string is invalid (after logging an error).

Example Usage


>>> get_file_class('file')
# Assuming File class is defined in __main__
>>> get_file_class('RFILE')
# Assuming RFile class is defined in __main__
>>> get_file_class('invalid')
# Output will also include a call to flag_system_error
None

are_strings_similar

The function uses a similarity metric to calculate the ratio between the target string and each string in the list. If an exact match is found, it returns a direct "Match Found" message. For strings that meet or exceed the similarity threshold, their percentage scores along with the corresponding string are included in the report. If no strings meet the threshold, the function notifies the caller that no similar strings were found. 

are_strings_similar(string_to_check, string_list, source_list='', threshold=0.7)

Parameters

Parameter Type Description
string_to_check str The string that will be compared against the list.
string_list list of str The collection of strings to compare with.
source_list str, optional A label or description representing the source list; this appears in the output report. Defaults to an empty string.
threshold float, optional The minimum similarity ratio (from 0 to 1) that qualifies a string as similar. The default is 0.7 (representing 70% similarity).

Returns

str: A report summarizing the similarity comparisons. It includes:
- An immediate "Match Found" message if an identical string is found.
- A detailed listing of the similarity percentages for strings that exceed the
threshold, along with a suggestion to verify the spelling.
- A notification if no strings in the list are similar enough.

Example Usage


>>> report = are_strings_similar("example", ["sample", "exemplary", "test"], "Test List")
>>> print(report)
Name does not match Test List!   (example)
----------------------------------------------------------------------
80.0% (sample)
90.0% (exemplary)
----------------------------------------------------------------------
Similar matches found.  Please check for correct spelling.

Framework Tools / File Operations

This group of functions provides utilities for reading and writing data to different file formats (text, JSON, CSV) and ensuring directory paths exist. They offer flexibility in how data is synchronized between in-memory Python objects (lists and dictionaries) and files, with options for overwriting or updating based on file existence and a specified priority. Additionally, a helper function (dumper) facilitates the creation of human-readable string representations of Python objects, and another (pathFinder) ensures directory paths are created if they don't exist.

Function/Class Description
updateTxtFromFile Updates a text file with the provided list of strings.
updateJson Updates a JSON file with the provided dictionary.
updateCsv Updates a CSV file with the provided dictionary.
dumper Build easy to read output for a given object.
pathFinder Ensures that a specified directory path exists.

updateTxtFromFile

Looks for a text file and creates a list of strings from the file if it does not exist. Otherwise, it updates the local list to match the file contents. 

updateTxtFromFile(local_list, filename, priority, file_class='file', encoding='utf-8', non_ascii_action='replace', replacement_char='*')

Parameters

Parameter Type Description
local_list list The list object that will get updated or update the file
filename str Path to the file
priority str Two modes:
(file) - if the file exists, it will overwrite the list. If the file does not exist, it will create the file from the list.
(local) - will always overwrite the file with the local list.
file_class str Valid args: 'file' or 'rfile'
encoding str The encoding to use when reading/writing the file. Defaults to 'utf-8'.
non_ascii_action str or None Action to take on non-ASCII characters.
Valid options: None, 'strip', 'replace'.
- None: No action (default).
- 'strip': Remove non-ASCII characters.
- 'replace': Replace non-ASCII characters with `replacement_char`.
replacement_char str The character to replace non-ASCII characters with if non_ascii_action is set to 'replace'. Defaults to '?'.

Returns

None

updateJson

This function checks for an existing JSON file at the specified path. Depending on the given priority, the function either updates the local dictionary with the file's contents (if the file exists) or overwrites the file with the local dictionary. It supports two priority modes:
- "file": If the file exists, the function updates the local dictionary with the contents of the file. If the file does not exist, it creates a new JSON file from the local dictionary.
- "local": The function always overwrites the file with the local dictionary, regardless of existing contents. 

updateJson(local_dict, filename, priority, sort_keys=False, file_class='file')

Parameters

Parameter Type Description
local_dict dict The dictionary that will either update the JSON file or receive updates from it.
filename str The path to the JSON file.
priority str A string indicating the update mode ('file' or 'local').
sort_keys bool, optional If True, the dictionary keys in the JSON file will be sorted alphabetically. Defaults to False.
file_class str, optional Indicates which file handling class to use ('file' or 'rfile'). Defaults to 'file'.

Returns

Exception: Propagates exceptions related to file access and JSON operations.

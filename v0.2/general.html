About SP Framework

Framework version: 1.0.2a

Introduction

The SP Framework is a comprehensive collection of helper modules and methods designed to make control systems easier to deploy. With a focus on clarity and accessibility, it's specifically crafted to be easily readable by entry to mid-level programmers.

Our framework offers a wide range of powerful capabilities including:

  • Multi-Panel Management - Seamlessly control and coordinate multiple interface panels
  • File Handling - Simplified operations for reading, writing, and managing configuration files
  • Device Handling - Streamlined interfaces for controlling various hardware components
  • Sandbox Creation - Safe testing environments for development and troubleshooting
  • And much more - Extensive utilities to support your control system needs

Designed with accessibility in mind, the SP Framework helps Day 1 programmers tackle advanced methodologies with greater ease and confidence. By abstracting complex operations into intuitive interfaces, we bridge the gap between beginner concepts and professional implementation.

Key Goals

  • Utilizing Easy-to-Read Files: Enables seamless configuration and scaling of control system programs for a facility. Ensures clarity and simplicity in program structures, reducing complexity for engineers and operators.
  • Increasing Standardization and Consistency: Promotes uniformity across different control system implementations. Reduces variability, making troubleshooting and updates more efficient.
  • Supporting O&M Teams for Adds, Moves, and Changes: Provides a structured approach for Operations and Maintenance (O&M) teams to perform modifications. Facilitates smooth transitions for system enhancements without disrupting operations.
  • Enhancing Security and Configuration Confidence: Strengthens system integrity by following structured methodologies. Minimizes risks of misconfigurations, ensuring a stable and secure deployment.
  • Ensuring Baseline Design for Functional Consistency: Establishes a foundation for consistent behavior across different system deployments. Simplifies system rollouts and updates by following predefined design principles.

Methodology

The standard methodology is utilizing a workflow that leverages the strengths of Python and the Extron ControlScript library. One of the key fundamentals is the use of standardized lists and dictionaries for instantiation, storage, and recall of core program components, objects, functions, and variables. The use of dictionaries specifically are employed to take advantage of their speed while avoiding unnecessary iteration.

Benefits

In addition to achieving the goals stated at the beginning of this document, the implementation of SP Framework has some very attractive benefits:

  • Dynamic Instantiation: Through basic edits in simple config files, you create a robust code base that grows with you.
  • Call custom global functions from any file without worry or order of operations errors
  • Change GUI attributes such as text, state, and value from any file - regardless of where they were instantiated.
  • Become the envy of all your friends Leverage networking capacity by soliciting jealousy amongst your peers
  • Robust troubleshooting tools are built in Trace Logs, trace across TCP, robust logging, and program SSH interfacing

Sandbox Management

The SP Framework provides a tiered approach to sandbox management, allowing different levels of users to interact with the system according to their expertise and responsibilities:

Tier III - Programmer

  • Create initial sandbox - Establish the foundation and architecture of the sandbox environment
  • Minimal follow up support - Provide specialized assistance for complex issues that cannot be resolved at lower tiers

Programmers are responsible for the initial creation and architecture of sandbox environments, setting up the framework that other tiers will work within. They focus on core development and only provide support for complex technical issues.

Certifications

Required: Extron Authorized Programmer

Recommended:

  • Extron AV Associate
  • Extron Network AV Specialist
  • Extron ProDSP Specialist

Tier II - AV Maintenance

  • Create / edit sandbox config files - Modify configuration settings to adapt the sandbox to specific requirements
  • Main follow up support - Serve as the primary support contact for ongoing sandbox management

AV Maintenance personnel handle the day-to-day management of sandbox environments, making necessary adjustments to configuration files and providing the main support channel for users. They bridge the gap between programmers and end users.

Certifications

Required: Extron AV Associate

Recommended:

  • Extron Network AV Specialist
  • Extron ProDSP Specialist
  • Extron Authorized Programmer

Tier I - AV Tech / End User

  • Basic Diagnostic tools available on the glass - Access to user-friendly diagnostic interfaces
  • Provide valuable feedback to AV Maintenance - Report issues and suggest improvements based on practical usage

End users and AV technicians interact with the sandbox through intuitive interfaces, using built-in diagnostic tools to identify and report issues. Their feedback is essential for continuous improvement of the system.

Certifications

Recommended: Extron AV Principles

Typical Workflow

The SP Framework implementation follows a structured workflow that guides users through the process of defining, configuring, and deploying control systems. This methodical approach ensures consistency, scalability, and maintainability across projects while accommodating the specific requirements of each facility. By following these steps, teams can efficiently develop robust control solutions that meet both technical specifications and user expectations.

1. Project Scope Definition

  • Define Room Type - Categorize spaces based on functionality and requirements
  • Determine Equipment Type - Identify the appropriate technology solutions for each space
→

2. Determine Equipment Footprint

  • Models of displays - Select appropriate display technologies and sizes
  • Cameras, Codecs - Choose video capture and processing equipment
  • Matrix Switchers - Determine signal routing and distribution needs
  • DSP's - Select audio processing solutions
  • Etc... - Additional equipment based on specific requirements
→

3. Determine Sandbox Walls

  • Min-Max displays - Define the range of supported display configurations
  • Min-Max Inputs - Establish input capacity requirements and limitations
  • Supported Classifications - Identify security and compliance requirements
→

4. Determine Control Footprint / Deltas

  • Cameras - Define camera control requirements and presets
  • Codecs, Blu-ray - Configure media playback and conferencing controls
  • Audio, Time Zone Clocks - Set up audio management and time displays
  • Messaging - Implement notification and communication systems
  • Etc... - Additional control elements as needed

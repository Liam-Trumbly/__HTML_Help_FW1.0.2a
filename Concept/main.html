Overview

The dynamic_text_buffer class is designed to manage dynamic text buffers with optional UI integration and color settings. It allows seamless updates to associated UI hosts and supports validation for color management.

Initialization

To create an instance of the dynamic_text_buffer class, use the following syntax:

buffer = dynamic_text_buffer(name="buffer_name", alias="optional_alias", UIHosts=["host1", "host2"], db=custom_db)
  • name: Unique identifier for the buffer (required).
  • alias: Alias for UI updates (optional).
  • UIHosts: List of UI hosts for updates (optional).
  • db: Database to register the buffer (default: var.dynamic_text_db).

Methods

update()

Triggers an update to push the current text buffer to the associated UI hosts.

self_register()

Registers the buffer in the specified database. If a duplicate record exists, the registration is skipped, and a debug message is logged.

Properties

text_buffer

Getter: Retrieves the current text buffer.
Setter: Updates the text buffer. Ensures the value is a string and triggers an update.

color

Getter: Retrieves the current text color.
Setter: Sets the text color. If the color is invalid, reverts to the default color and logs a debug message.

Examples

buffer = dynamic_text_buffer(name="example", alias="exampleAlias", UIHosts=["host1"], db={})

buffer.text_buffer = "Hello, World!"
print(buffer.text_buffer)  # Outputs: Hello, World!

buffer.color = "red"
print(buffer.color)  # Outputs: red
